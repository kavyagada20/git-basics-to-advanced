# 🐞 GitHub Issues

---

## 📌 Overview

GitHub Issues are used to **track tasks, bugs, enhancements, and discussions** in a repository.

👉 They act as a lightweight project management tool.

---

## 🔧 What are Issues?

An Issue represents:
- A bug 🐞
- A feature request ✨
- A task 📋
- A discussion 💬

---

## 🚀 Creating an Issue

### Steps:
1. Go to your GitHub repository  
2. Click on **Issues** tab  
3. Click **New Issue**  
4. Add:
   - Title
   - Description
5. Submit  

---

## 🏷️ Issue Components

### 🔹 Title
Short summary of the issue

### 🔹 Description
Detailed explanation

### 🔹 Labels
Categorize issues:
- bug
- enhancement
- documentation

### 🔹 Assignees
Person responsible for resolving the issue

### 🔹 Milestones
Group issues into goals

---

## 🧪 Example Issue

```text
Title: Fix login button not working

Description:
The login button does not respond when clicked.
Steps to reproduce:
1. Open login page
2. Click login button
3. Nothing happens
🔄 Issue Workflow
Create Issue → Assign → Work → Close Issue
🔗 Link Issues with PR

You can link issues in PRs:

Fixes #12
Closes #15

👉 Automatically closes issue when PR is merged

📝 Comment on Issues
Add updates
Discuss solutions
Ask questions
✅ Close Issue
Manually click Close issue
OR via commit/PR message
🧠 Best Practices
Write clear and descriptive titles
Provide steps to reproduce bugs
Use labels properly
Keep discussions relevant
⚠️ Common Mistakes
Vague issue descriptions
No reproduction steps
Not assigning issues
Ignoring issue updates
🎯 Summary
Issues track tasks and bugs
Improve team collaboration
Integrate with pull requests
